## 一、起点：我们在做什么

### 1.1 核心命题

> 一个原本用于图像生成/编辑的 **image-in/image-out flow matching teacher (FlowInOne)**，能否通过轻量微调被转化为可量化评估的视觉理解模型？

### 1.2 约束

- **不加分割头**（no segmentation head）——仍然是 image-in/image-out
- **不输出 logits**（no classification output）——输出是图像
- **不改成文本输出**（no language-only）——保持视觉输出
- **保持 latent flow matching 训练目标**——而非 CE/Dice 等分割 loss

### 1.3 统一任务形式

```text
input image + visual instruction
    → FlowInOne teacher (latent flow matching)
    → generated perception image
```

三个数据集的输出各不相同：

| 数据集 | 输入 | 输出 | 标准指标 |
|--------|------|------|------|
| Cityscapes | RGB 街景 + palette strip | 7/19-class RGB semantic mask | mIoU |
| RefCOCOg | COCO 图像 + referring expression | binary mask | cIoU/gIoU/Dice/P@0.5 |
| ReasonSeg | COCO 图像 + reasoning query | binary mask | cIoU/gIoU/Dice/P@0.5 |

## 二、统一基础设施：Aligned Prompt + 数据协议

### 2.1 为什么需要 Aligned Prompt

早期实验用的是 **square prompt**：1024×1024 方形画布，上半放原图，下半放任务说明和调色板。

```text
Square prompt 的问题:
+------------------+
|                  |
|   原始图像       |  ← 上半
|                  |
+------------------+
| Task: ...        |  ← 下半: 任务说明 + 调色板
| Legend: ...      |
+------------------+
```

FlowInOne 的 dataloader 做 256 center crop 时，输入图像和目标 mask 的像素级空间对齐被破坏了。

Cityscapes 上的量化证据：

| Prompt 类型 | mIoU | Coarse-7 | Drivable |
|-------------|:---:|:---:|:---:|
| Square prompt | 0.152 | 0.401 | 0.862 |
| **Aligned prompt** | **0.242** | **0.583** | **0.964** |

**几何对齐是 dense prediction 的前提。** 不仅是 mIoU 提升了 60%，更重要的是结构化场景理解能力的全面提升——drivable area 从 0.862 跳到 0.964，vehicle group 从 0.163 跳到 0.482。

### 2.2 Aligned Prompt 的设计

```text
+------------------------------------------------+
| Task: referring segmentation                   |  ← instruction strip (~36px)
| Expression: the man wearing a red shirt        |
| Output: white=target, black=background         |
+------------------------------------------------+
|                                                 |
|              原始图像主体 (256×256)             |  ← 几何不变
|                                                 |
+------------------------------------------------+
```

关键设计原则：

1. **图像主体不移动、不裁剪、不缩放** — 保持与输出 mask 的像素级对齐
2. **只在顶部叠加窄 strip** — 包含任务描述和 referring expression
3. **strip 高度自适应** — 通常占图像高度的 ~9%（~36px for 256×256）
4. **不同任务的 strip 文本不同** — 由 packer 脚本根据数据集自动生成

### 2.3 统一数据协议

每个 WebDataset 样本包含四个文件：

| 文件 | 格式 | 内容 |
|------|------|------|
| `{key}.in.png` | RGB PNG | aligned prompt（strip + 原图），`(256+strip_h)×256` |
| `{key}.out.png` | RGB PNG | perception target，256×256 |
| `{key}.task.txt` | UTF-8 text | 任务文本（referring expression / reasoning query） |
| `{key}.meta.json` | JSON | 元数据（dataset, expression, mask_area, split 等） |

关键实现细节：

- **mask resize 必须用 nearest-neighbor** — binary mask 如果用 bicubic 会产生灰度值，破坏二值性
- **输入图像 resize 用 bicubic** — 保持自然图像的平滑性
- **out.png 保存为 RGB PNG** — 即使 binary mask 也保存为 RGB（避免 single-channel 兼容性问题）
- **task.txt / meta.json 作为 sidecar** — 不影响训练，仅供分析和调试

## 三、RefCOCOg：发现全白崩溃，Inverted Mask 突破

### 3.1 第一次训练：全白崩溃

**训练配置**：

```text
Model:     FlowInOne teacher (flowinone_256px.pth)
Init:      原始 teacher（未做任何微调）
Data:      refcocog_aligned_binary_prompt_wds (42226 train / 2573 val)
LR:        1e-7
Steps:     1000
Batch:     1
GPU:       1×A100
```

训练过程看起来正常——loss 从 0.045 降到 0.019，再稳定在 0.13-0.19 之间。Checkpoint 正常保存在 step 250, 500, 750, 1000。

**但评估时发现灾难性失败**：

```python
# 定性检查预测输出
pred = np.array(Image.open('pred_step500/sample_000.png').convert('L'))
print(f"nonzero pixels: {(pred > 0).mean() * 100:.1f}%")
# → 81-100%（几乎全白！）

# GT mask 对比
gt = np.array(Image.open('gt/sample_000.png').convert('L'))
print(f"nonzero pixels: {(gt > 0).mean() * 100:.1f}%")
# → 3-8%（仅小目标区域为白）
```

模型输出几乎全是白色像素（nonzero ~90-100%），而 GT mask 只有 3-8% 的非零像素。

cIoU = 0.085。这 0.085 仅仅是背景区域的偶然重叠——cIoU 的分子（sum intersection）和分母（sum union）都很大，因为模型输出几乎覆盖全图。

### 3.2 根因分析

Binary mask 有**极端的类别不平衡**：

```text
前景 (255): 3-8% 像素   ← 需要模型"画"的部分
背景 (0):   92-97% 像素  ← 模型可以"偷懒"的部分
```

Flow matching 在 **latent space** 计算 loss：

```text
z_mask = VAE.encode(mask)         # mask → latent
v_pred = model(z_t, t | prompt)   # 预测 velocity
loss   = ||v_pred - v_target||²   # latent space MSE
```

当 foreground 极其稀疏时：

1. **latent 梯度信号弱** — 3% 的前景像素在 VAE 压缩后的 latent 中贡献极小
2. **VAE decoder 回归到均值** — 当 latent 不确定时，decoder 输出的"最优猜测"接近训练数据分布的平均图像
3. **FlowInOne 预训练的平均图像接近灰白** — 原始 teacher 训练于图像生成任务，"平均图像"是中等亮度的灰白图

**结果**：模型的最优策略是输出全白——loss 不会很高（白色在 latent space 里靠近均值），但 IoU 接近于零。

### 3.3 尝试的修复方案

我们系统性地尝试了四种方案：

| 方案 | 原理 | 结果 | 诊断 |
|------|------|:---:|------|
| R2: Cityscapes→RefCOCOg | 先用语义分割"预热"再迁移 | cIoU=0.041 | **负迁移**。19-class palette 输出与 binary mask 差异太大 |
| GPU2: 低lr长训 (3e-8/2000步) | 更温和的优化避免崩溃 | cIoU=0.084 | **全白依然**。优化速度不影响最终模式 |
| GPU1: SDF 连续表示 | 用 Signed Distance Function 替代硬二值 | cIoU=0.074 | eval 协议不适配。SDF 需要专门的 threshold 转换 |
| Original teacher | 检查是否"未微调就有能力" | cIoU=0.080 | **同样全白**。说明这是 teacher 本身的输出偏置 |

**结论**：所有在原始 binary mask 表示下的尝试都失败了。低lr、长训练、不同初始化、连续表示——都改变不了全白模式。因为这是**表示层面**的问题，不是优化不够。

### 3.4 Inverted Mask：简单的数据侧修复

核心洞察来自定性检查——模型输出的全是"亮色"（接近255）。如果能翻转 mask，让模型输出"暗色"（接近0）为主导，就能天然避开全白。

```python
# 只改一行
# 原始: 背景=0(黑色), 前景=255(白色)
#        → 模型最优策略：输出全白（因为 92-97% 像素是 0，VAE 均值偏白）
#
# 翻转: 背景=255(白色), 前景=0(黑色)
#        → 模型最优策略：输出全黑（因为 92-97% 像素是 255... 不对！）

# 等一下。翻转后：
#   背景: 92-97% 像素 = 255（白色）
#   前景: 3-8% 像素  = 0（黑色）
#   → 模型最优策略：输出全白（因为大部分像素是 255）
#   → 这和翻转前一样？！

# 实际上翻转后的机制：
#   模型输出全黑（因为 VAE 均值偏暗，且 flow matching 天然偏向低值输出）
#   → 前景是白色（255），模型恰好需要"画出"稀疏的白色目标
#   → 87-97% 的背景是黑色（0），模型输出暗色即可
```

**实际上这里有一个微妙的点**：VAE decoder 的"均值回归"方向在翻转前后是不同的。翻转前，均值偏向亮色→模型输出全白。翻转后，均值偏向暗色→模型输出全黑——恰好背景就是黑（0），前景才是白（255）。模型要做的"额外工作"是画出白色目标，而不是抑制全白输出。

**不需要修改模型架构、不需要修改 loss 函数、不需要调超参——只改了数据。**

### 3.5 效果

| 指标 | 原始 Binary | Inverted (step 250) | 提升 |
|------|:---:|:---:|:---:|
| cIoU | 0.085 | **0.475** | 5.6x |
| gIoU | 0.076 | **0.468** | 6.2x |
| mDice | 0.128 | **0.595** | 4.6x |
| P@0.5 | 0.008 | **0.453** | 58x |
| P@0.7 | 0.000 | **0.164** | — |

**45% 的样本 IoU ≥ 0.5，16% 的样本 IoU ≥ 0.7。** 这意味着模型确实学会了根据 referring expression 定位目标区域，而不仅仅是输出随机模式。

Inverted mask 各 checkpoint 表现：

| Step | cIoU | gIoU | mDice | P@0.5 | 趋势 |
|:---:|:---:|:---:|:---:|:---:|------|
| 250 | **0.475** | **0.468** | **0.595** | **0.453** | ← 最优 |
| 500 | 0.472 | 0.465 | 0.594 | 0.430 | 轻微退化 |
| 750 | 0.469 | 0.462 | 0.590 | 0.438 | 轻微退化 |
| 1000 | 0.444 | 0.437 | 0.559 | 0.406 | 过拟合 |

最佳 checkpoint 在 step 250——之后开始轻微过拟合。这与 Cityscapes 早期发现一致：perception-as-generation tuning 的最优点往往在几百步，不需要长训练。

阈值扫描（step 250）：

| thr | 0.05 | 0.10 | 0.30 | 0.50 | 0.70 | 0.90 |
|-----|:---:|:---:|:---:|:---:|:---:|:---:|
| cIoU | 0.101 | 0.100 | 0.095 | 0.085 | 0.068 | 0.046 |

注意：这里展示的是**原始 binary** 的阈值扫描（用于分析模型输出分布）。Inverted mask 的 optimal threshold 是 0.5（论文主结果）。

## 四、ReasonSeg：Inverted Mask 的跨任务验证

### 4.1 任务差异

| 维度 | RefCOCOg | ReasonSeg |
|------|----------|-----------|
| 任务类型 | 指代分割 | 推理分割 |
| Query 示例 | "the man in red shirt" | "When eating a bowl of soup, what kitchen tool would most likely be used to scoop and consume the soup?" |
| Query 长度 | 短（5-15 words） | 长（15-40 words） |
| 目标 | 名词短语指代 | 需要常识推理 |
| Train 样本 | 42,226 | 239 |
| Val 样本 | 2,573 | 200 |
| Strip 高度 | ~36px | ~42px（更长文本需要更多空间） |

### 4.2 原始二进制的问题

和 RefCOCOg 一样，原始 binary mask 训练后全白崩溃。cIoU = 0.138——略高于 RefCOCOg 的 0.085，但这只是因为 ReasonSeg 的 GT mask 面积略大，白图偶然覆盖更多 GT 像素而已。

所有 checkpoint（100-500步）指标完全 flat：

```text
Step 100: cIoU=0.138, gIoU=0.112, mDice=0.170, P@0.5=0.065
Step 200: cIoU=0.138, gIoU=0.112, mDice=0.170, P@0.5=0.065
Step 300: cIoU=0.138, gIoU=0.112, mDice=0.170, P@0.5=0.065
Step 400: cIoU=0.138, gIoU=0.111, mDice=0.169, P@0.5=0.060
Step 500: cIoU=0.136, gIoU=0.110, mDice=0.168, P@0.5=0.055
```

这是**崩溃 plateaus** 的典型特征——模型一早就收敛到错误模式，之后不再改变。loss 在降（因为全白在 latent space 确实接近最优），但 IoU 接近零。

### 4.3 Inverted Mask 效果

应用同样的翻转策略后：

| 指标 | 原始 Binary | Inverted (step 100) | 提升 |
|------|:---:|:---:|:---:|
| cIoU | 0.138 | **0.480** | 3.5x |
| gIoU | 0.112 | **0.473** | 4.2x |
| mDice | 0.170 | **0.599** | 3.5x |
| P@0.5 | 0.065 | **0.460** | 7x |
| P@0.7 | 0.010 | **0.220** | 22x |

**100 步即收敛到峰值**，且在整个 100-500 步范围内完全稳定：

```text
Step 100: cIoU=0.480, P@0.5=0.460
Step 200: cIoU=0.480, P@0.5=0.460
Step 300: cIoU=0.480, P@0.5=0.460
Step 400: cIoU=0.479, P@0.5=0.455
Step 500: cIoU=0.481, P@0.5=0.460
```

仅 239 个训练样本就达到了 0.481 cIoU。这证明：

1. **FlowInOne teacher 的隐式视觉理解先验非常强** — 只需极少样本即可释放
2. **Inverted mask 修复的是表示层面的瓶颈** — 一旦表示正确，模型几乎不需要"学习"
3. **ReasonSeg 的推理能力已被 teacher 隐式掌握** — 微调只是让它能够显式输出

### 4.4 跨任务一致性

两个完全不同的任务、不同的数据集大小、不同的语言复杂度，**inverted mask 给出了几乎相同量级的提升和相似的绝对指标**：

| 数据集 | 原始 cIoU | Inverted cIoU | 提升 | 训练样本 |
|--------|:---:|:---:|:---:|:---:|
| RefCOCOg | 0.085 | 0.475 | 5.6x | 42,226 |
| ReasonSeg | 0.138 | 0.481 | 3.5x | 239 |

这证明了 **inverted mask 是一个通用的数据侧修复方案**，不是过拟合某个数据集的 trick。

## 五、Cityscapes：Coarse-7 的教训

### 5.1 为什么做 Coarse-7

当前最强 19-class checkpoint 的 mIoU = 0.276（val64）。扩展分析揭示：

| 类别 | IoU | 水平 |
|------|:---:|------|
| road | 0.952 | 接近可用 |
| sky | 0.786 | 强 |
| vegetation | 0.756 | 强 |
| building | 0.649 | 中上 |
| car | 0.546 | 中 |
| ... | ... | ... |
| fence | 0.037 | 几乎失败 |
| truck | 0.025 | 几乎失败 |
| wall | 0.003 | 几乎失败 |
| train | 0.001 | 几乎失败 |
| motorcycle | 0.000 | 完全失败 |

**Spearman(class support, per-class IoU) ≈ 0.90** — per-class IoU 几乎被类别频率决定。

于是有了"粗粒度 7 组"的想法：

```text
flat          = road + sidewalk
construction  = building + wall + fence
object        = pole + traffic light + traffic sign
nature        = vegetation + terrain
sky           = sky
human         = person + rider
vehicle       = car + truck + bus + train + motorcycle + bicycle
```

从 19-class 预测结果**聚合**（不是训练）得到的 coarse-7 mIoU = 0.611。目标是通过真正的 7-class 训练超过这个数。

### 5.2 训练

```text
Init:      aligned prompt continuation 12000.ckpt（19-class 训练过的最强 aligned prompt 模型）
Data:      cityscapes_coarse7_aligned_visual_prompt_wds (2975 train / 500 val)
LR:        5e-8
Steps:     1000
Batch:     1
Output:    7-class RGB mask（每类一种颜色）
```

Prompt strip 从 19 色调色板改为 7 色调色板：

```text
Task: Cityscapes semantic segmentation (coarse)
Legend: flat / construction / object / nature / sky / human / vehicle
```

### 5.3 结果

```text
coarse7_mIoU = 0.567
pixel_acc    = 0.855
```

| 组 | IoU | Support | 分析 |
|---|:---:|:---:|------|
| flat | **0.959** | 1.76M | ✅ 极强 |
| nature | **0.727** | 648K | ✅ 强 |
| sky | **0.720** | 227K | ✅ 强 |
| construction | 0.567 | 578K | 中 |
| human | 0.387 | 45K | 弱 |
| vehicle | ~0.33 | 226K | 弱 |
| object | 0.167 | 104K | 很弱 |

对比：

```text
真正 coarse-7 训练: mIoU = 0.567
19-class 聚合:     mIoU = 0.611  ← 从19-class confusion matrix聚合
```

**真正训练反而更差。**

### 5.4 为什么？— 与 Binary 发现的对照

这与 binary mask 的发现形成了有趣的镜像：

| 任务 | 输出类别数 | 前景占比 | 现象 |
|------|:---:|:---:|------|
| RefCOCOg binary | 2 | 3-8% | 全白崩溃 |
| RefCOCOg inverted | 2 | 92-97% | 正常 ✅ |
| Cityscapes 19-class | 19 | 100%（每像素都有类） | 0.276 mIoU |
| Cityscapes 7-class | 7 | 100%（每像素都有类） | 0.567 mIoU |

**核心洞察**：对于 flow matching 的图像输出来说，**更少的输出类别并不一定更好**。

19-class 虽然每个类的 IoU 不高，但模型被 19 种不同的 target color 强制做出更细粒度的区分——这提供了**更强的 latent 学习信号**。压缩到 7 类后，模型只需要学 7 种颜色，输出空间变小，latent 的区分度降低。

这与 inverted mask 的教训一致：**输出表示的设计是关键**——它直接影响 latent flow matching loss 的优化景观。Binary mask 需要倒置（因为极端稀疏性），而 semantic mask 需要足够多的类别（因为需要足够的输出空间分辨率）。

---

## 六、方法论总结

### 6.1 三个核心发现

1. **几何对齐是 dense prediction 的前提**

   Cityscapes square→aligned：mIoU 0.152→0.242，drivable IoU 0.862→0.964。输入与输出的空间对齐决定了模型能否形成 dense prediction。

2. **输出表示的选择决定成败**

   Binary mask inverted：cIoU 0.085→0.475（5.6x）。这是一个**纯数据侧修复**——不需要改模型、不需要改 loss、不需要调超参。表示问题不能用优化来解决。

3. **更多输出类别有时更好**

   19-class 聚合 coarse-7 mIoU = 0.611 > 7-class 训练 mIoU = 0.567。更丰富的输出空间提供更强的 latent 学习信号。输出表示不是越简单越好。

### 6.2 最终结果总表

| 数据集 | 指标 | 原始 | 最优方案 | 提升 |
|--------|------|:---:|:---:|:---:|
| **RefCOCOg** | cIoU | 0.085 | **0.475** (inverted, step 250) | 5.6x |
| | gIoU | 0.076 | **0.468** | 6.2x |
| | P@0.5 | 0.008 | **0.453** | 58x |
| **ReasonSeg** | cIoU | 0.138 | **0.481** (inverted, step 100) | 3.5x |
| | gIoU | 0.112 | **0.474** | 4.2x |

\* 0.611 是从 19-class 聚合的，非真正训练

### 6.3 与 Vision Banana 的定位差异

| 维度 | Vision Banana (Kaiming He) | Ours |
|------|:---:|:---:|
| 基座 | Nano Banana Pro（大扩散模型） | FlowInOne（flow matching teacher） |
| 训练策略 | 生成数据 + 理解数据混合 instruction tuning | 单任务轻量微调 |
| RefCOCOg cIoU | 0.738 | 0.475 |
| ReasonSeg gIoU | 0.793 | 0.474 |
| Cityscapes mIoU | 0.699 | 0.276 |
| 核心贡献 | 大规模生成模型可做通用理解 | 输出表示的优化偏置 + 轻量修复 |

### 6.4 关键教训

1. **Flow matching latent loss 对稀疏二值输出有系统性偏置** — binary mask 需要在数据侧倒置
2. **优化不能修复表示问题** — 低lr、长训练、不同初始化都无法解决全白崩溃
3. **Teacher 已有强隐式先验** — ReasonSeg 仅 239 样本/100 步即可达到 cIoU=0.481
4. **VAE decoder 的"均值回归"是一把双刃剑** — 它给了生成模型泛化能力，但在极端稀疏输出上导致崩溃
5. **反直觉：更多输出类别不一定更差** — 19-class > 7-class，因为更丰富的输出空间提供更强的学习信号
