# 交接：三个数据集的理解能力微调

本文档记录 RefCOCOg、ReasonSeg、Cityscapes 三个数据集在 FlowInOne teacher 上的 perception-as-generation 微调全过程。下一个会话通过读取本文档即可完全了解当前状态、文件位置、实验结果和下一步计划。

---

## 0. 一句话总结

我们在 FlowInOne (image-in/image-out flow matching teacher) 上对三个视觉理解数据集做了 perception-as-generation 微调。发现 **binary mask 存在"全白崩溃"问题**——模型输出 ~90%+ 白色像素而非稀疏二值 mask。通过简单的 **mask 反转**（白底黑目标）彻底解决，在两个 binary 数据集上获得 3.5-5.6x 提升。

---

## 1. 整体架构与核心概念

### 1.1 任务形式

```text
input:  aligned visual prompt (原图 + 顶部 instruction strip)
        → FlowInOne teacher (flow matching)
        → perception image (semantic mask / binary mask)
```

```text
aligned prompt 格式:
+------------------------------------------------+
| Task: referring segmentation                   |  ← instruction strip
| Expression: the man wearing a red shirt        |
| Output: white=target, black=background         |
+------------------------------------------------+
|                                                 |
|              原始图像主体 (256×256)             |
|                                                 |
+------------------------------------------------+
```

### 1.2 关键约束

- **不加分割头** — 仍然是 image-in/image-out flow matching
- **不输出 logits** — 输出是 RGB/binary 图像
- **保持 latent flow matching 训练目标** — 非 CE/Dice 等分割 loss
- **几何对齐** — 输入图像主体与输出 mask 空间对齐（aligned prompt 核心创新）

### 1.3 三个数据集的统一形式

| 数据集 | 输入 | 输出 | 指标 |
|--------|------|------|------|
| Cityscapes | RGB 街景 + palette strip | 7/19-class RGB semantic mask | mIoU |
| RefCOCOg | COCO 图像 + referring expression | binary mask (inverted) | cIoU/gIoU/Dice/P@0.5 |
| ReasonSeg | COCO 图像 + reasoning query | binary mask (inverted) | cIoU/gIoU/Dice/P@0.5 |

---

## 2. 核心发现：全白崩溃与 Inverted Mask

### 2.1 问题

首次 RefCOCOg 训练完成后，定性检查发现：

```
R1 binary mask 输出: nonzero=81-100% (几乎全白)
GT mask:             nonzero=3-8%    (仅小目标区域为白)
```

**根本原因**：binary mask 前景(255)仅占 3-8% 像素，背景(0)占 92-97%。Flow matching 在 latent space 训练，对稀疏前景的梯度信号太弱，VAE decoder 收敛到输出"平均图像"——接近全白。

### 2.2 解决方案

```python
# 翻转 mask: 255→0, 0→255
# 效果: 前景变白(255)，背景变黑(0)
# 模型自然输出暗色主导的图像，恰好需要"画出"白色目标区域
inverted = 255 - original_mask
```

翻转后：背景 ~95% 像素值为 0，模型输出天然以暗色为主，避免了全白崩溃。

### 2.3 效果

| 数据集 | 原始 cIoU | Inverted cIoU | 提升 |
|--------|:---:|:---:|:---:|
| RefCOCOg | 0.085 | **0.475** | 5.6x |
| ReasonSeg | 0.138 | **0.481** | 3.5x |

**这是一个通用的数据侧修复方案**，在两个不同任务上一致有效，无需修改模型/损失函数。

for split in ['train', 'val']:

    out_dir = Path(f'teacher_understanding_eval/data/refcocog_aligned_inverted_prompt_wds/{split}')
    out_dir.mkdir(parents=True, exist_ok=True)
    for tar_path in sorted(src_dir.glob('pairs-*.tar')):
        out_tar = out_dir / tar_path.name
        with tarfile.open(tar_path) as src, tarfile.open(out_tar, 'w') as dst:
            for member in src.getmembers():
                data = src.extractfile(member).read()
                if member.name.endswith('.out.png'):
                    img = Image.open(io.BytesIO(data)).convert('L')
                    inverted = 255 - np.array(img)
                    buf = io.BytesIO()
                    Image.fromarray(inverted).save(buf, format='PNG')
                    data = buf.getvalue()
                ti = tarfile.TarInfo(member.name)
                ti.size = len(data)
                dst.addfile(ti, io.BytesIO(data))
"

#### 评估计算的核心公式

```python
# 逐样本
intersection = sum(pred & gt)
union = sum(pred | gt)
iou = intersection / union
dice = 2 * intersection / (pred_area + gt_area)

# 聚合
cIoU = sum(all_intersections) / sum(all_unions)   # cumulative IoU
gIoU = mean(all_ious)                               # mean per-sample IoU
mDice = mean(all_dices)                             # mean per-sample Dice
P@0.5 = fraction of samples with IoU >= 0.5
```

## 7. 关键教训与设计原则

1. **Binary mask 表示选择至关重要** — 白底黑目标的"暗色主导"范式 >> 黑底白目标的"亮色稀疏"范式。这是 flow matching latent loss 对稀疏二值输出的优化偏置导致的。

2. **Aligned prompt 几何对齐是基础** — 输入图像与输出 mask 必须在空间上对齐，否则模型无法形成 dense prediction。

3. **Cityscapes→RefCOCOg 是负迁移** — 19-class palette 输出干扰了 binary mask 学习。跨任务微调需要谨慎。

4. **Low lr / 长训练不能解决全白问题** — 模型从 step 250 到 2000 始终崩溃在相同模式。这是表示层面的问题，不是优化不够。

5. **ReasonSeg 收敛极快** — 仅 239 样本、100 步即饱和。说明 teacher 已有强视觉理解先验，只需轻量微调即可释放。

6. **SDF 需要专门评估协议** — 连续距离场不能直接用 binary threshold 评估。可能需要将 SDF 转换回 binary mask 后再算指标。
